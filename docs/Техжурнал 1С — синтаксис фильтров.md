# ✅ Где задаются фильтры

Фильтры задаются в файле:

📌 `logcfg.xml`

Обычно лежит тут:

- Windows:  
    `C:\Program Files\1cv8\conf\logcfg.xml`
    
- Linux:  
    `/opt/1cv8/x86_64/<version>/conf/logcfg.xml`
    

---

# ✅ Общий вид фильтра

```xml
<event>
    <eq name="Event" value="EXCP"/>
</event>
```

Это значит: логировать только события типа `EXCP`.

---

# 🔥 Основные операторы фильтрации

## 1) `eq` — равно

```xml
<eq name="Event" value="DBMSSQL"/>
```

Только SQL-события.

---

## 2) `ne` — не равно

```xml
<ne name="Usr" value="admin"/>
```

Все пользователи кроме admin.

---

## 3) `like` — шаблон (подстрока)

```xml
<like name="Descr" value="%deadlock%"/>
```

Все события, где в описании есть “deadlock”.

---

## 4) `in` — список значений

```xml
<in name="Event">
    <value>EXCP</value>
    <value>DBMSSQL</value>
</in>
```

Логировать только ошибки и SQL.

---

## 5) `and` / `or`

```xml
<and>
    <eq name="Event" value="EXCP"/>
    <like name="Descr" value="%access%"/>
</and>
```

Ошибки, содержащие “access”.

---

```xml
<or>
    <eq name="Event" value="EXCP"/>
    <eq name="Event" value="ELOG"/>
</or>
```

Ошибки или события журнала.

---

## 6) `not`

```xml
<not>
    <eq name="Event" value="DBMSSQL"/>
</not>
```

Логировать всё, кроме SQL.

---

# ✅ Полезные поля (name=)

Вот самые важные параметры:

|Поле|Значение|
|---|---|
|`Event`|Тип события (`EXCP`, `DBMSSQL`, `CALL`…)|
|`Usr`|Пользователь|
|`AppName`|Имя приложения (тонкий клиент, сервер)|
|`Pid`|ID процесса|
|`Session`|UUID сессии|
|`Descr`|Текст ошибки/описание|
|`Context`|Контекст выполнения|

---

# 🎯 Примеры реально полезных фильтров

---

## Логировать только ошибки (EXCP)

```xml
<event>
    <eq name="Event" value="EXCP"/>
</event>
```

---

## Только медленные запросы

```xml
<event>
    <eq name="Event" value="DBMSSQL"/>
    <like name="Descr" value="%duration%"/>
</event>
```

---

## Только конкретный пользователь

```xml
<event>
    <eq name="Usr" value="Иванов"/>
</event>
```

---

## Всё кроме фоновых заданий

```xml
<event>
    <ne name="AppName" value="BackgroundJob"/>
</event>
```

---

## Сессия, которая “висит”

```xml
<event>
    <eq name="Session" value="c7d3..."/>
</event>
```

---

# 🔥 Максимально полезный шаблон (боевой)

```xml
<event>
    <or>
        <eq name="Event" value="EXCP"/>
        <eq name="Event" value="ELOG"/>
        <eq name="Event" value="DBMSSQL"/>
    </or>
</event>
```

Ошибки + журнал + SQL.

---

# ⚠️ Важно

Техжурнал может **убить производительность**, если включить всё подряд.

Правило:

✅ включаем точечно  
❌ никогда не включаем без фильтров на проде
